## P0 — CRITIQUE (5 issues)

### P0-1 : Escalade de privileges horizontale — routes Sophia

| Fichier | Description |
|---------|-------------|
| `api/sophia/contracts/route.ts` | `organizationId` vient des query params sans controle. Un utilisateur peut acceder aux contrats d'autres organisations. |
| `api/sophia/organizations/route.ts` | `orgId` libre dans les params → acces a n'importe quelle organisation. |
| `api/sophia/sites/route.ts` | `organizationId` non valide → acces aux sites d'autres organisations. |

**Remediation** : Forcer `organizationId` depuis `process.env.SOPHIA_ORGANIZATION_ID` ou valider que l'utilisateur appartient a l'organisation demandee.

### P0-2 : IDOR sur sites/select

| Fichier | Description |
|---------|-------------|
| `api/sophia/sites/select/route.ts` | `resourceId` du body non valide et aucune verification d'appartenance. Tout utilisateur authentifie peut modifier n'importe quelle ressource. |

**Remediation** : Valider `resourceId` (format UUID/id_dpl) et verifier que la ressource appartient a un BDC accessible.

### P0-3 : Docker container en root

| Fichier | Description |
|---------|-------------|
| `Dockerfile` | Le conteneur s'execute en tant que root. |

**Remediation** :
```dockerfile
RUN addgroup -g 1001 -S nodejs && adduser -S nextjs -u 1001
USER nextjs
```

## Points Positifs

### Securite
- Requetes SQL 100% parametrees (hors LIMIT/OFFSET)
- JWT avec algorithme explicite HS256
- Secrets via variables d'environnement
- Rate limiting sur login et refresh
- Regles de mot de passe (longueur, majuscule, minuscule, chiffre)
- `requireAuth()` sur toutes les routes
- Whitelist des champs PATCH (`BDC_ALLOWED_FIELDS`, `RESOURCE_ALLOWED_KEYS`)
- Headers securite (X-Frame-Options, X-Content-Type, Referrer-Policy, Permissions-Policy)

### Qualite
- TypeScript strict active, zero erreurs `tsc --noEmit`
- Typage explicite des retours GraphQL (`executeGraphQL<T>`)
- Interfaces claires (BonDeCommande, RessourceDpl, Client, etc.)
- Validation UUID centralisee avec support suffixe `-NNNN`
- Pagination securisee avec bornes (1-200)
- Filtres et tris valides contre des listes blanches

### Performance
- Pool MySQL singleton via `globalThis` (survit au hot reload)
- `Promise.all` pour les stats (3 requetes paralleles)
- JOINs et sous-requetes GROUP BY pour eviter la duplication
- SWR avec cache et revalidation automatique

### Architecture
- Separation claire : auth / db / sophia / validation
- Middleware Next.js pour l'auth globale
- Client Sophia singleton avec refresh token automatique
- Multi-stage Docker build (image legere)
- Healthcheck configure dans docker-compose
